# Convenciones de nomenclatura
- Formato Variables y funciones: camelCase
- Archivos: kebab-case
# Formato de código:
Se prohibe el uso de estilos en línea. 
# Estrategia de branches:
- No se hacen commits en main. 
- Código estable y listo para entrega. 
# Estrategia de commits: 
- Los commits se hacen solo para cambios significativos.
- Comentarios claros que mejoren la comprensión. 
- Variables con nombres significativos. 
## Los mensajes de commit siguen la convención: 
### Tipos de commit:
- New: Archivos nuevos
- Updated: Actualización de archivo. 
- Style: para cambios de formato. 
- Fix: corrección de errores o ajustes pequeños.
- Docs: para cambios en documentación. 
- Incomplete: para guardar cambios pero aún no se ha completado el documento.
- Develop: integración de nuevas funciones antes de subir a main.
